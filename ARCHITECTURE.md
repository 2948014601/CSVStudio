# CSV Studio 项目架构设计

## 1. 当前设计基线

根据最新需求，这个项目的定位已经收敛为“专业级 CSV 表格查看器”，重点是：

- 大规模 CSV 的稳定打开与滚动浏览
- 工业灰白配色，界面语言参考 Excel 一类桌面生产力软件
- 明确可见的加载进度条，而不是无反馈等待
- 以表格为核心，不保留数据图像导出相关主流程

## 2. 当前技术选型

当前工作区可直接使用：

- `PySide6`：桌面 UI 主框架
- `pandas`：样本探测、列类型推断、预览分析
- `pyarrow`：后续可作为高性能中间层候选

当前首版采用：

`PySide6 + 分块索引 + QTableView 虚拟模型 + QThread 后台加载`

这样做的优点：

- 不依赖额外安装
- 桌面端 UI 和表格滚动体验稳定
- 可以先显示预览，再后台建立完整索引
- 更适合做真正可用的大文件 CSV 浏览器

## 3. 分层结构

### 3.1 表现层 Presentation

文件：

- `csv_studio/main_window.py`
- `csv_studio/styles.py`
- `csv_studio/widgets/title_bar.py`

职责：

- 自定义无边框窗口壳层
- 工业灰白主题 QSS
- 顶部标题栏、主工具栏、左侧资源栏、中央主表格区、右侧详情面板、底部状态条
- 可见进度条与加载状态提示

### 3.2 数据层 Infrastructure

文件：

- `csv_studio/services/csv_service.py`

职责：

- 编码探测
- 分隔符探测
- CSV 样本读取
- 列类型推断
- 分块偏移索引构建
- 块级缓存读取

### 3.3 视图模型层 ViewModel / Qt Model

文件：

- `csv_studio/models/csv_table_model.py`

职责：

- 把分块数据以 `QAbstractTableModel` 形式提供给 `QTableView`
- 提供按单元格读取、列样本读取、标题格式化、颜色渲染
- 提供预览片段导出能力

### 3.4 后台加载层 Worker

文件：

- `csv_studio/workers/csv_loader.py`

职责：

- 先完成轻量预览
- 在后台线程中继续构建完整索引
- 持续回传进度百分比与状态文本
- 在失败时回退到“仅预览”状态

## 4. 当前大文件策略

### 4.1 设计思路

对于几十万到几百万行 CSV，当前版本不一次性整表读入内存，而是采用两段式：

1. 先读取有限样本，完成：
   - 表头识别
   - 列类型推断
   - 缺失值 / 重复样本分析

2. 再建立块偏移索引：
   - 每 `2000` 行记录一个字节偏移
   - 表格滚动时按块加载
   - 最近访问块进入缓存
   - 按文件扫描进度实时更新进度条

### 4.2 优点

- 内存占用可控
- 打开大文件时界面不假死
- 用户能明确看到加载进展
- 表格滚动体验更接近真正的数据浏览器

## 5. 当前视觉方向

### 5.1 为什么使用工业灰白主题

用户当前最直接的需求是：

- 界面更像成熟工业软件
- 避免花哨深色风格
- 参考 Excel 的清晰层级和稳定观感

因此本轮重构采用：

- 灰白底色与白色内容面板
- 更克制的圆角和阴影
- 中性灰边框与表头层级
- Excel 风格绿色作为主强调色

### 5.2 视觉关键点

- 外层工作区：浅灰
- 主表格面板：白色
- 工具栏 / 标题栏：浅灰分层
- 表头：更深一级的中性灰
- 强调色：`#217346`

## 6. 功能演进路线

### 阶段 A：当前已完成

- 工业灰白主题重构
- 去掉原生标题栏白边
- 可见加载进度条
- 预览先展示 + 后台建立索引
- 大文件表格浏览与滚动

### 阶段 B：专业数据工作台

- 真正排序 / 筛选 / 去重
- 取消加载与后台任务管理
- 更细粒度的分块进度提示
- 数据质量规则面板

### 阶段 C：可编辑数据产品

- 单元格编辑与撤销 / 重做
- 数据清洗流水线
- 保存 / 另存为
- 多文件标签工作区
